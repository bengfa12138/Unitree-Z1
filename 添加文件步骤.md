# 新添加执行文件步骤

## 文件说明

| 文件 | 用途 |
|---|---|
| `highcmd_development_annotated.cpp` | 官方 `highcmd_development.cpp` 的中文注释版，适合理解官方轨迹控制逻辑 |
| `z1_custom_waypoints.cpp` | 多路点自定义轨迹示例，适合直接改 `p1/p2/p3/p4` 来控制机械臂运动 |

## 添加文件步骤

把文件复制到 SDK 示例目录：

```bash
cp /root/work/z1_trajectory_examples/z1_custom_waypoints.cpp \
   /root/work/unitree_z1/z1_sdk/examples/z1_custom_waypoints.cpp
```
将新的执行文件添加到
```bash
/root/work/unitree_z1/z1_sdk/examples
```
文件夹下

编辑 `CMakeLists.txt`：

```bash
cd /root/work/unitree_z1/z1_sdk
nano CMakeLists.txt
```

找到：

```cmake
set(EXAMPLES_FILES
    examples/highcmd_basic.cpp
    examples/highcmd_development.cpp
    examples/lowcmd_development.cpp
    examples/lowcmd_multirobots.cpp
)
```

加一行：

```cmake
    examples/z1_custom_waypoints.cpp
   （格式为examples/文件名）
```

变成：

```cmake
set(EXAMPLES_FILES
    examples/highcmd_basic.cpp
    examples/highcmd_development.cpp
    examples/lowcmd_development.cpp
    examples/lowcmd_multirobots.cpp
    examples/z1_custom_waypoints.cpp
)
```

重新编译：

```bash
docker start unitree-z1-noetic
docker exec -it unitree-z1-noetic bash
cd /root/work/unitree_z1/z1_sdk/build
cmake ..
make -j$(nproc)
```

运行：

```bash
./z1_custom_waypoints
```

## 运行顺序

保持三个终端顺序：

```text
终端 1：roslaunch unitree_gazebo z1.launch
终端 2：./sim_ctrl
终端 3：./highcmd_development 或 ./z1_custom_waypoints
```

先允许 Docker 容器访问图形界面：

```bash
xhost +local:root
```

需要同时开 3 个终端。

### 终端 1：启动 Gazebo 和 Z1 模型

主机新开终端：

```bash
docker start unitree-z1-noetic
docker exec -it unitree-z1-noetic bash
```

如果权限不足：

```bash
sudo docker start unitree-z1-noetic
sudo docker exec -it unitree-z1-noetic bash
```

进入容器后：

```bash
source /opt/ros/noetic/setup.bash
source /root/work/unitree_ws/devel/setup.bash
roslaunch unitree_gazebo z1.launch
```

正常情况会打开 Gazebo，并显示 Z1 机械臂模型。

### 终端 2：启动 Z1 仿真控制桥

主机新开第二个终端：

```bash
docker exec -it unitree-z1-noetic bash
```

进入容器后：

```bash
source /opt/ros/noetic/setup.bash
source /root/work/unitree_ws/devel/setup.bash
cd /root/work/unitree_z1/z1_controller/build
ls -lh sim_ctrl
./sim_ctrl
```

终端 2 不要关闭。它负责连接 SDK 和 Gazebo。

### 终端 3：运行 SDK 控制示例

主机新开第三个终端：

```bash
docker exec -it unitree-z1-noetic bash
```

进入容器后：

```bash
cd /root/work/unitree_z1/z1_sdk/build
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/root/work/unitree_z1/z1_sdk/lib:/root/work/unitree_z1/z1_controller/lib
ls -lh highcmd_basic
./（文件名不要后缀）
```

如果一切正常，Gazebo 中的 Z1 会开始响应 SDK 示例命令。
